# Architecture Decision Records — jannahclinic

ADRs are the only Level-1 authority (see
`DOCS-AUTHORITY-AND-CONFLICT-RESOLUTION.md`). Any redesign of architecture,
structure, navigation, or system direction MUST go through a new ADR or a
formal update to an existing one.

## Lifecycle

`DRAFT` → `ACCEPTED` → `ACTIVE` → `SUPERSEDED`

- An ADR governs only while `ACCEPTED`/`ACTIVE` **and** listed in the
  Canonical Decision Registry.
- Superseding an ADR requires a new ADR that names the one it replaces and a
  registry update (both rows).

> ADRs use the lifecycle statuses above (DRAFT/ACCEPTED/ACTIVE/SUPERSEDED), not the document status vocabulary in DOCS-AUTHORITY-AND-CONFLICT-RESOLUTION.md §4. An ACCEPTED/ACTIVE ADR listed in the registry is Level-1 canonical authority by definition.
